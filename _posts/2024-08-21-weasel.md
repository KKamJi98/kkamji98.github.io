2024년 8월 21일 Weasel 프로젝트를 성공적으로 마무리 했습니다.  

Github: <https://github.com/Team-S5T1>  

Weasel 프로젝트는 Bedrock Sonnet 3.5 Model을 사용한 문제풀이 서비스로 Frontend 2명, Backend 2명, Infra 2명 총 6인으로 진행하였고, 인프라를 담당했습니다. 처음 팀장으로써 프로젝트를 진행하며 내가 잘 할수 있을지 걱정이 되었지만 팀원들 덕분에 직면한 문제를 수월하게 해결할 수 있었던 것 같습니다. 이번 포스트에서는 Weasel 프로젝트에 대해 소개하고, 트러블 슈팅과 느낀점에 대해 다뤄보도록 하겠습니다.

## 주요 기능

로그인 (Spring Security, OAuth)  
문제풀이  
프롬프트 히스토리  
응답 스트리밍  

![한글 문제 풀이](/assets/img/weasel/korean_exam.webp)

![영어 문제 풀이](/assets/img/weasel/english_exam.webp)

## 기술 스택

Backend => Spring Boot
Frontend => React
IaC => Terraform
Deploy => Amazon EKS, S3, Route53, CloudFront
Container Registry => ECR
CI/CD => Jenkins, ArgoCD
AI => AWS Badrock(Claude 3.5 sonnet)
Monitoring => Prometheus, Grafana

## 인프라 아키텍처

> 인프라는 AWS 클라우드를 기반으로 구축했습니다. 트래픽이 과도하게 발생했을 때 장애를 최소화하기 위해 EKS를 사용했으며 HPA를 사용해 Pod Autoscaling과 Karpenter를 사용해 Node Autoscaling을 구현했습니다. 뿐만 아니라 보안을 위해 Secrets Manager를 사용해 중요 환경변수의 노출을 방지하려 노력했고, Terraform을 모듈을 생성하고 이를 사용해 인프라를 구축하면서 자동화, 버전 관리, 일관성을 추구했습니다.
